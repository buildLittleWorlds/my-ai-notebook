Methods for Reading AI Texts (Atomic Procedures)

What This Page Is (and Isn’t)

  • Is: a toolbox of small, testable actions you can perform on any AI text to generate evidence (counts, maps, notes).
  • Isn’t: a stance or goal-driven playbook. For that, see Reading Strategies.
  • Output: concrete artifacts you can reuse across strategies: tables of signals, topic maps, opposition charts, world sketches, task logs.

How to Use These Methods

  1. Pick 2–4 methods aligned to your question.
  2. Run them sequentially (signals → maps → probes), saving artifacts as you go.
  3. Carry the artifacts into a strategy (e.g., Forensic, Derrida, Ricoeur) to make claims with warrants.

Signals & Scans

Method A: Transition Frequency Scan

Goal: Surface formulaic scaffolds and pacing tics that indicate distributional generation.

Steps:

  1. Underline transitional phrases (e.g., “In conclusion,” “Moreover,” “On the other hand”) across 500–800 words.
  2. Count occurrences; compute per-100-words rate.
  3. Note clustering (e.g., list bias, paragraph starters).

Output: a small table of phrases → counts → notes on placement.

Pairs with strategies: Forensic (signal weighting), Pragmatist (readability issues).

Method B: Register Drift Check

Goal: Detect mixing of formal/colloquial styles characteristic of LLMs under weak constraints.

  1. Mark sentences with colloquialisms/slang and those with high formality.
  2. Flag abrupt shifts within 2–3 sentences.
  3. Annotate any genre clashes (e.g., academic voice → marketing speak).

Output: margin flags + a brief drift summary.

Method C: Logical Profile Pass

Goal: Identify argumentation gaps (assertion → example → inference) and circularity.

  1. Label every paragraph as claim, evidence, example, or bridge.
  2. Highlight unsupported claims or orphaned examples.
  3. Note where terms shift meaning (definition drift).

Output: a paragraph-level legend + list of gaps.

Mapping & Modeling

Method D: Topic Migration Map

Goal: Chart how the text wanders across nodes (topics, motifs).

  1. List keywords per paragraph (3–5 each).
  2. Draw a simple flow (P1→P2→…); mark jumps and returns.
  3. Infer a governing tendency (e.g., breadth-first sampling, example chaining).

Output: a linear map of topic movement + a one-sentence tendency hypothesis.

Method E: Hermeneutic Circle for Systems

Goal: Replace author-psychology with system + corpus modeling.

  1. Local → Global: from a paragraph’s features, hypothesize the text’s generative pattern.
  2. Global → Local: test if later paragraphs conform; note breakdowns.
  3. Revise the hypothesis; repeat.

Output: a short memo stating the hypothesized pattern and counterexamples.

Stress Tests & Probes

Method F: Binary-Opposition Map

Goal: Prepare for deconstructive pressure by listing key binaries and their hierarchy.

  1. Extract candidate binaries (e.g., natural/artificial, author/machine).
  2. Mark which term is privileged in context.
  3. Find passages where the hierarchy flips or frays.

Output: a two-column chart with notes on instability points.

Method G: Iterability Probe

Goal: Observe meaning drift when repeating or relocating a key line (after Derrida).

  1. Choose a pivotal sentence; move it to 2–3 new contexts in the text.
  2. Record how sense and force change.
  3. Optionally, re-prompt the model with the sentence to test portability across outputs.

Output: a short table: context → observed drift.

Reference: Derrida, “Signature Event Context.” UCI Hydra.

Method H: Value-Claim Map

Goal: List the value judgments the text advances; ask who benefits/loses (preps genealogy).

  1. Underline evaluatives (“good,” “should,” “innovative,” etc.).
  2. Annotate implied beneficiaries and exclusions.
  3. Group claims by affected stakeholder.

Output: a table of claims → winners/losers.

World & Use

Method I: World-of-the-Text Sketch

Goal: Describe the world the text projects (Ricoeur): agents, actions, norms, possibilities.

  1. List agents and roles the text presupposes.
  2. List actions it makes imaginable and legitimate.
  3. State the kind of reader it invites you to become.

Output: a one-paragraph sketch + bullets of permissible actions.

Method J: Task Attempt Log

Goal: Evaluate practical fitness by attempting what the text prescribes.

  1. Attempt the task exactly as written.
  2. Log failure points, ambiguities, error costs.
  3. Revise minimal wording; re-attempt; record delta.

Output: a short log with pass/fail and friction notes.

“As If” Heuristic (Use Carefully)

Treat the text as if it had an authorial center when this clarifies ethics/rhetoric—but do not import biography. Drop the heuristic when it misleads.

Method Artifacts You Can Reuse

  • Topic map & system memo (from D, E)
  • Opposition & iterability tables (from F, G)
  • Value-claim and world sketch (from H, I)
  • Task log (from J)
